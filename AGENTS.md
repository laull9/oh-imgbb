## 代码风格
- 使用 snake_case 命名函数和变量
- 使用 PascalCase 命名结构体和枚举
- 使用 UPPER_SNAKE_CASE 命名常量
- 分层合理
- 每个函数和结构体前有一行简短中文注释
- 代码复用清晰
- 错误处理使用 anyhow 库，返回 Result 类型
- 使用 async/await 处理异步操作

## 注意事项

- 使用中文注释和文档
- 避免过度设计，保持代码简洁
- 每个函数和struct上有一行简短的注释，核心函数和trait接口需要更详细的注释
- 框架测试完备，直接写在实现文件后面，不用故意区分tests层
- 不要单文件打全部，单文件非测试代码不得超过500行，平行逻辑合理拆分和复用
